# MySQL

## 乐观锁（数据库自己实现的）
    数据有版本号。每次更新的时候，数据库根据版本号来判断，只有版本号相同才更新。
    数据库自己实现的
## 悲观锁（程序自己实现的）
    每次请求数据的时候，需要在用户程序内加锁。
    共享锁（read lock） 排他锁() 是悲观锁的不同实现。
    要使用悲观锁，需要关闭autocommit模式。

## 按权限来划分
### 共享锁 (read lock 读锁)
    可以并发读。但是不能修改。只有数据上的 所有的 共享锁都解除了。才能写。
### 排他锁（write lock 写锁）
    锁定后，其他线程不能对记录进行，读写操作。直到当前线程释放锁。
